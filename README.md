# Web-приложение для анализа аудиосигналов

Это full-stack веб-приложение, которое позволяет пользователю загружать аудиофайл и получать его визуальное представление в виде волновой формы и спектрограммы.

> Проект разработан студентами 3 курса **НИТУ МИСИС** в рамках программы **Цифровой Кафедры** по направлению **"Кибербезопасность и защита данных"**.

## Технологический стек

*   **Backend**: Python, FastAPI, Uvicorn
*   **Frontend**: React, TypeScript, Vite
*   **Оркестрация**: Docker, Docker Compose
*   **Веб-сервер/Прокси**: Nginx

## Быстрый старт с Docker

Для запуска всего проекта требуется только установленный **Docker** и **Docker Compose**.

### 1. Клонирование репозитория
```sh
git clone <URL_вашего_репозитория>
cd <название_папки_проекта>
```

### 2. Конфигурация бэкенда
Перед первым запуском необходимо создать файл конфигурации для бэкенда.

Создайте файл `.env` в папке `backend` (`backend/.env`) и скопируйте в него следующее содержимое:

```env
# backend/.env
SECRET_KEY=generate_your_own_strong_secret_key_here
STORAGE_DIR=/app/storage
```
*   **`SECRET_KEY`**: Замените `generate_your_own_strong_secret_key_here` на любую длинную и случайную строку.
*   **`STORAGE_DIR`**: Оставьте `/app/storage`, это путь внутри Docker-контейнера.

### 3. Сборка и запуск контейнеров
Выполните следующую команду в корневой папке проекта:

```sh
docker-compose up --build -d
```
*   `--build`: принудительно пересобирает образы на основе Dockerfile.
*   `-d`: запускает контейнеры в фоновом (detached) режиме.

### 4. Доступ к приложению
После успешного запуска:
*   **Веб-интерфейс** будет доступен в браузере по адресу: `http://localhost:3000`
*   **Документация API (Swagger)** бэкенда доступна по адресу: `http://localhost:8000/docs`

## Остановка приложения
Чтобы остановить и удалить все запущенные контейнеры, выполните команду:
```sh
docker-compose down
```
